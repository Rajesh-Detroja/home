How to initialize the terminal

How to initialize the terminal

Open a terminal. Depending on the defaults of your system it may look like the image below:

What you really want is something that looks like the following:

On a Mac you go to the “Terminal->Preferences” and set the “Novel” color scheme as default, then increase the font sizes. Now open a new terminal. It should look nice and clean.

On Windows you need to right click the panel and set colors for foreground and background as well as font sizes. The color scheme I use has the background hexcolor of #DEDBC6 and text color of black #000000.

Make the prompt nicer

The “prompt” is the little symbol at the beginning of the line.

See how the unix prompt in our example above shows the name of the user ialbert the name of the computer yolo, and the current path ~ in green. I found this information to be most useful long term, once you become more familiar with using Unix and once you log into other computing systems from the machine, these little pieces of information become incredibly useful. You always want to know which computer are you logged into, and where exactly are you located.

To set up your prompt the same way mine is you can open a Terminal and run the following:

curl http://data.biostarhandbook.com/install/bash_profile.txt >> ~/.bash_profile
curl http://data.biostarhandbook.com/install/bashrc.txt >> ~/.bashrc

These commands will update your so-called “shell” (see later) to behave more consistently. If you want the more details and the nitty gritty of what is going on read the Setting the Bash profile page.

Avoid running the terminal initialization commands more than once!

The new settings won’t apply right away to the current terminal window. To have them applied, you will need to either close then re-open the terminal or type:

source ~/.bash_profile

What does the prompt show?

The bash prompt is that the text/symbol that appears on each line. If you follow the instruction above the prompt is been customized to give you information on the context that you are working on.

Initially, when you only work on a single system and in a single directory the information seems unnecessary and perhaps overbearing. As soon as you gain experience, perhaps log into a remote machine it will be your first line of defense in avoiding some of the most dangerous mistakes you can make: running a command on the wrong computer or in the wrong directory.

Thus the prompt is set up to continually, even obnoxiously, remind you of the following:

  • What computer and what user are you logged in as.
  • What directory are you currently in.

Cherish this information :-) on day it will save your bacon.
